## Contract Architecture

### `RWAOracle.sol`

The primary entry point. Accepts a Groth16 proof, verifies it on-chain, prevents replay via nullifiers, and credits the caller's collateral balance (TEE-encrypted on Sapphire).

### `Verifier.sol`

Auto-generated by `snarkjs zkey export solidityverifier`. Contains the pairing-based Groth16 verifier. **Do not manually edit this file.**

---

## ABI — Key Functions

### `submitRWAProof`

```solidity
function submitRWAProof(
    uint256[8] calldata proof,      // Groth16 (A[2], B[4], C[2]) packed
    uint256 stateRoot,              // Source chain Merkle root (public)
    uint256 nullifierHash,          // Poseidon(addr, trapdoor) — replay token
    uint256 minCollateral,          // Threshold value proven in ZK circuit
    address assetContract           // Source chain RWA token address
) external
```

Reverts with `"ZK Proof Invalid"` if verification fails.  
Emits `RWAProofSubmitted(account, assetContract, minCollateral)` on success.

### `getActiveCollateral`

```solidity
function getActiveCollateral() external view returns (uint256)
```

Returns the caller's total verified collateral. On Oasis Sapphire, this mapping is **TEE-encrypted** — only `msg.sender` can read their own value via `eth_call`.

## How to Submit a Proof (ethers.js v6)

```typescript
import { ethers } from "ethers";
import RWAOracleABI from "./abi/RWAOracle.json";

const provider = new ethers.JsonRpcProvider("https://testnet.sapphire.oasis.io");
const signer = await provider.getSigner();

const oracle = new ethers.Contract(
  "0x2Df7658D5E57ed05D6F634fD7d73b334ADEc179A",
  RWAOracleABI,
  signer
);

// proof from SnarkJS groth16.prove()
const proofPacked = [
  proof.pi_a[0], proof.pi_a[1],
  proof.pi_b[0][0], proof.pi_b[0][1],
  proof.pi_b[1][0], proof.pi_b[1][1],
  proof.pi_c[0], proof.pi_c[1],
];

const tx = await oracle.submitRWAProof(
  proofPacked,
  publicSignals[0],  // stateRoot
  publicSignals[2],  // nullifierHash
  publicSignals[1],  // minCollateral
  "0xRWATokenAddressOnSourceChain"
);

await tx.wait();
console.log("Collateral verified:", tx.hash);
```

```bash

npx hardhat run scripts/deploy.js --network sapphire_testnet
```

```bash

  ../../packages/upe-core-circuits/build/rwa_shield_final.zkey \
  src/Verifier.sol
```

## Trust Model

> **Deploy only to Oasis Sapphire.**

On standard EVM chains (Ethereum, Polygon, Arbitrum), the `nullifiers` and `activeCollateral` mappings are readable via `eth_getStorageAt` by anyone. Sapphire's Trusted Execution Environment (TEE) encrypts all contract storage — making the mappings actually private. The Groth16 proof verification is cryptographically sound on any EVM; the storage confidentiality guarantee is Sapphire-specific.
